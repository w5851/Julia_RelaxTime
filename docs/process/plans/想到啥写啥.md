在 server_full.jl 设计 PNJL 异方程接口前，先封装 Trho/Tmu 为“长任务 + CSV streaming”形式，或提供更轻量的单点 Solver（减少文件输出和扫描循环）。

临时灵感
- `data/` 目录可以沿用 “raw / processed / outputs” 模式：`data/raw/pnjl/seeds/*.csv` 保存粗网格种子，视为不可修改的原始离线结果；`data/processed/` 代表中间结果用于后续插值或合并；`data/outputs/` 存放实时求解的 CSV。这样便于区分再生数据与一次性资产。
- 针对 T-MU-RHO-XI 空间无需全遍历，可采用稀疏采样：先选一组代表性温度 (ΔT≈20 MeV)、化学势 (Δμ≈50 MeV)、密度 (0.2 步长) 与 ξ (0.2 步长)，生成有限集合；在线求解时做最近邻或简单插值即可，减少离线计算成本。
    稀疏采样可能可以按照类似“对角线取点”的思路？不知道可行性如何。
    具体的采样方法参考docs/reference/domain-knowledge/参数空间预计算点的取样方法.md

- 实现 PNJL 主线时，可在 `src/pnjl/` 内分层：`parameters/`（TOML 解析）、`seeds/`（缓存）、`solvers/`（单点/扫描）、`routes/`（服务器适配）。所有模块遵循 `module Foo; ... end` 结构、export 具体函数，并在 `src/pnjl/PNJL.jl` 统一 include，保持与现有 `src/relaxtime/*` 的组织风格一致。
- 对热力学部分的积分可以用Julia_test项目中测试的节点方式提高精度
- pnjl模块待实现的功能：Tmu空间扫描，Trho空间扫描，根据Trho得结果绘制rho-mu图，根据rho-mu图的s形判断相图中的稳态与亚稳态分界点，根据s形消失的位置判断临界点位置，根据Tmu的结果绘制P-mu图，根据P-mu图判断不同温度下一阶相变线的位置，最终绘制相图
- pnjl模块待实现的功能2： 计算涨落（目前只实现了求解热力学势和序参量），计算涨落需要对解方程得到的热力学量作各阶导数计算，可以使用自动微分实现，注意要对满足方程的热力学量求导，即要将解方程放在自动微分的计算图中
- pnjl模块待实现的功能3： 实现各向异性PNJL模型，目前只实现了各向异性下的热力学势计算和序参量求解，还需要实现各向异性PNJL模型下的Tmu空间扫描和Trho空间扫描，以及根据扫描结果绘制相图
- pnjl模块待实现的功能4： 实现前端UI，将不同的功能模块分开，用户可以通过前端选择不同的功能模块进行计算和绘图
