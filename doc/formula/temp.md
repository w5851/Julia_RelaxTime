```markdown
## 性能要求
- 核心算法: < 1ms 每百万元素
- 内存分配: 零动态分配在热路径中
- 缓存命中率: > 90% 在典型工作负载

## 代码审查清单

### 性能审查要点
- [ ] 热路径中无动态内存分配
- [ ] 分支预测友好
- [ ] 内存访问模式连续
- [ ] 无冗余计算
- [ ] 编译器优化提示适当
- [ ] 输入验证在调试模式外最小化

### 可读性保持
- [ ] 复杂优化有注释说明
- [ ] 性能关键部分有基准测试
- [ ] 算法选择理由文档化

**注意**: 本指南强调性能优化，但所有优化必须基于性能分析数据。避免过早优化，确保优化确实解决性能瓶颈。
```

## 主要改进说明

### Markdown 文档的改进：
1. **结构更清晰**：将语言要求单独列出，突出重要性
2. **格式统一**：保持了原有的检查清单格式，便于人工阅读
3. **内容完整**：保留了所有原始要求，无内容丢失

### YAML 配置的特点：
1. **结构化组织**：将相关要求分组，便于 Agent 按类别访问
2. **键值明确**：每个要求都有明确的键名，避免歧义
3. **易于扩展**：可以方便地添加新的性能指标或审查要点
4. **机器友好**：Agent 可以通过标准 YAML 解析器准确获取所有规则

这种双格式方案既满足了人类工程师的可读性需求，又为 AI Agent 提供了结构化的、无歧义的配置数据。
