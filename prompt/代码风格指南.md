# 高性能计算代码风格指南

## 语言要求
- 使用 Julia 语言编写

## 性能验证要求

### 1. 基准测试
- 所有性能关键函数必须包含基准测试、API文档与单元测试

### 2. 性能回归测试
```markdown
## 性能要求
- 核心算法: < 1ms 每百万元素
- 内存分配: 零动态分配在热路径中
- 缓存命中率: > 90% 在典型工作负载
```

## 代码审查清单

### 性能审查要点
- [ ] 热路径中无动态内存分配
- [ ] 分支预测友好
- [ ] 内存访问模式连续
- [ ] 无冗余计算
- [ ] 编译器优化提示适当
- [ ] 输入验证在调试模式外最小化

### 可读性保持
- [ ] 复杂优化有注释说明
- [ ] 性能关键部分有基准测试
- [ ] 算法选择理由文档化

**注意**: 本指南强调性能优化，但所有优化必须基于性能分析数据。避免过早优化，确保优化确实解决性能瓶颈。
